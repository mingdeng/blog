MongoDB在Linux上的安装得按相应操作系统版本选择对应的安装包

发表于 2017-07-27

查看linux版本，登录到服务器执行 lsb_release -a ,即可列出所有版本信息,例如:

注:这个命令适用于所有的linux，包括Redhat、SuSE、Debian等发行版。
我的系统是CenOS6.8，按相应操作系统版本选择对应的安装包如centos6 rhel6选择RHEL 6 Linux 64ibt，即mongodb-linux-x86_64-rhel62-3.4.6.tgz这个安装包。

今天遇到了一个坑，就是在CentOS6上安装CentOS7环境下的版本，即mongodb-linux-x86_64-rhel70-3.4.6.tgz安装包，运行mongod报错libc.so.6: versionGLIBC_2.14’ not found`。

这个问题非常不好解决，原因是系统的glibc版本太低，软件编译时使用了较高版本的glibc引起的。glibc是GNU发布的libc库，即c运行库。glibc是linux系统中最底层的api，几乎其它任何运行库都会依赖于glibc。
查看glibc版本命令

所以CentOS6用mongodb rhel62安装包版本，glibc不会出现这种问题。
